# 3.2.1. Bridge (Estrutural)

## 1. Introdução

O **Bridge** é um padrão estrutural do GoF que **separa uma abstração de sua implementação** para que ambas possam variar independentemente. No projeto MadDev, o Bridge desacopla a **abstração da UI** (o *quê* mostrar — HP, score, tempo) da **implementação de renderização** (o *como* mostrar — `CanvasRenderer` em nós visuais ou `DebugRenderer` no console). Trocar o renderer no Inspector muda a forma de exibir sem alterar a abstração.

### Quando usar

- Quando se quer evitar uma vinculação rígida entre uma classe (o "quê") e sua implementação concreta (o "como").
- Quando ambas — abstração e implementação — precisam ser estendidas via subclasse de forma independente.
- Quando é útil trocar a implementação em runtime ou via configuração (Inspector do Godot).

### Estrutura

```
HUDAbstraction (Control)  ──@export──▶  UIRenderer (Resource)
                                              ▲
                                              │
                          ┌───────────────────┴───────────────────┐
                          │                                       │
                  CanvasRenderer                          DebugRenderer
                  (ProgressBar/Label)                     (print no console)
```

## 2. Implementação no Projeto

### Arquivos

| Arquivo | Papel no padrão |
|---|---|
| `jogo/scripts/ui/renderers/ui_renderer.gd` | **Implementor** — interface abstrata de renderização |
| `jogo/scripts/ui/renderers/canvas_renderer.gd` | **ConcreteImplementor** — renderiza em nós visuais (Label, ProgressBar) |
| `jogo/scripts/ui/renderers/debug_renderer.gd` | **ConcreteImplementor** — renderiza em `print()` no console |
| `jogo/scripts/ui/hud_abstraction.gd` | **Abstraction** — define o que mostrar; delega ao renderer |

Os arquivos vivem na branch `game`.

### UIRenderer (Implementor)

```gdscript
## Bridge Pattern — Implementação (Implementor). Interface abstrata de renderização do HUD.
extends Resource
class_name UIRenderer

## Recebe as referências de nós da cena coletadas pela HUDAbstraction.
func bind(_targets: Dictionary) -> void:
    pass

# Operações virtuais — sobrescritas pelas implementações concretas
func render_hp(_value: float, _max_value: float) -> void: pass
func render_score(_value: int) -> void: pass
func render_time(_seconds: float) -> void: pass
```

### CanvasRenderer (ConcreteImplementor visual)

```gdscript
extends UIRenderer
class_name CanvasRenderer

var _hp_bar: ProgressBar = null
var _hp_label: Label = null
var _score_label: Label = null
var _time_label: Label = null

func bind(targets: Dictionary) -> void:
    _hp_bar = targets.get("hp_bar")
    _hp_label = targets.get("hp_label")
    _score_label = targets.get("score_label")
    _time_label = targets.get("time_label")

func render_hp(value: float, max_value: float) -> void:
    if _hp_bar != null:
        _hp_bar.max_value = max_value
        _hp_bar.value = value
    if _hp_label != null:
        _hp_label.text = "HP: %d / %d" % [int(value), int(max_value)]

func render_score(value: int) -> void:
    if _score_label != null:
        _score_label.text = "Score: %d" % value

func render_time(seconds: float) -> void:
    if _time_label != null:
        var minutes: int = int(seconds) / 60
        var secs: int = int(seconds) % 60
        _time_label.text = "Tempo: %02d:%02d" % [minutes, secs]
```

### DebugRenderer (ConcreteImplementor textual)

```gdscript
extends UIRenderer
class_name DebugRenderer

func render_hp(value: float, max_value: float) -> void:
    print("[HUD] HP: %d / %d" % [int(value), int(max_value)])

func render_score(value: int) -> void:
    print("[HUD] Score: %d" % value)

func render_time(seconds: float) -> void:
    print("[HUD] Tempo: %.1fs" % seconds)
```

### HUDAbstraction (Abstraction)

```gdscript
extends Control
class_name HUDAbstraction

# Bridge — referência para a Implementação (atribuída via Inspector)
@export var renderer: UIRenderer

@onready var _hp_bar: ProgressBar = $VBoxContainer/HealthRow/HealthBar
@onready var _hp_label: Label = $VBoxContainer/HealthRow/HealthLabel
@onready var _score_label: Label = $VBoxContainer/ScoreLabel
@onready var _time_label: Label = $VBoxContainer/TimeLabel

func _ready() -> void:
    if renderer != null:
        renderer.bind({
            "hp_bar": _hp_bar,
            "hp_label": _hp_label,
            "score_label": _score_label,
            "time_label": _time_label,
        })
    SignalBus.player_health_changed.connect(_on_player_health_changed)
    SignalBus.score_changed.connect(_on_score_changed)
    # ... outras conexões do Observer

func _render_hp(value: float, max_value: float) -> void:
    if renderer != null:
        renderer.render_hp(value, max_value)
```

A `HUDAbstraction` **não conhece os nós internos do renderer** — ela apenas envia as referências dos nós da cena via `bind()` e chama `render_*()`. A escolha do renderer é feita no Inspector via `@export var renderer: UIRenderer`.

## 3. Funcionamento

Para trocar a implementação de renderização — por exemplo, durante desenvolvimento para usar `DebugRenderer` que imprime no console em vez de atualizar nós visuais — basta:

1. Abrir a cena `hud.tscn` no editor do Godot.
2. Selecionar o nó `HUDAbstraction`.
3. No Inspector, no campo **Renderer**, trocar de `CanvasRenderer` para `DebugRenderer` (ou vice-versa).

**Nenhuma linha de código muda.** A `HUDAbstraction` continua chamando `renderer.render_hp(...)`, `renderer.render_score(...)`, etc. — só que agora os métodos delegam para uma implementação diferente.

### Benefícios

- **Decisão tardia:** a escolha do renderer pode ser feita no Inspector, sem recompilação.
- **Múltiplas implementações em paralelo:** pode existir um `CanvasRenderer` para release, um `DebugRenderer` para desenvolvimento, e (futuramente) um `WebRenderer` para builds web — todos coexistindo.
- **Extensibilidade independente:** adicionar uma nova operação na `HUDAbstraction` ou um novo renderer concreto não afeta o outro lado.
- **Testabilidade:** o `DebugRenderer` substitui o renderer visual em testes automatizados sem depender de cenas.

## 4. Metodologia

A construção desta documentação seguiu três etapas, tomando como fonte primária o código da branch `game`:

1. **Identificação dos papéis Abstraction/Implementor.** Leitura de `hud_abstraction.gd` para mapear a referência `@export var renderer: UIRenderer` e os métodos de alto nível (`_render_hp`, `_render_score`); leitura de `ui_renderer.gd` para confirmar a interface virtual.

2. **Análise das implementações concretas.** Comparação entre `canvas_renderer.gd` (manipula `ProgressBar`/`Label` via referências injetadas em `bind()`) e `debug_renderer.gd` (apenas `print()` no console) para evidenciar que ambas obedecem à mesma interface.

3. **Trace do ciclo de vida.** Verificação no `_ready()` da abstração de que o `bind()` é chamado uma vez (passando referências de nós da cena), e que a partir daí toda renderização passa pelo `renderer`.

**Ferramentas utilizadas:** Godot 4 (leitura do código e troca do renderer no Inspector), Git (leitura da branch `game`), Markdown + Docsify (apresentação).
