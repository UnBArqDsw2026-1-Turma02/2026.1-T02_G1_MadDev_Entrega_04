# 3.1.1. Factory Method (Criacional)

## 1. Introdução

O **Factory Method** é um padrão criacional do GoF que **delega a criação de objetos a um método dedicado**, permitindo desacoplar quem usa o objeto de quem decide qual classe concreta instanciar. No projeto MadDev, a Factory centraliza a criação de inimigos: em vez de `PackedScene.instantiate()` espalhado pelo código, há uma única função estática que recebe um tipo (`StringName`) e devolve o inimigo já configurado.

### Quando usar

- Quando a decisão de "qual classe instanciar" depende de um parâmetro (tipo, estado, configuração).
- Quando se quer adicionar novos tipos sem modificar o código que consome a fábrica.
- Quando o tipo concreto não deve vazar para o código de gameplay.

## 2. Implementação no Projeto

### Arquivos

| Arquivo | Papel |
|---|---|
| `jogo/scripts/enemies/enemy_factory.gd` | A Factory propriamente dita — método estático `create(type)`. |
| `jogo/scenes/enemies/enemy.tscn` | Cena do inimigo `basic`. |
| `jogo/scenes/enemies/enemy_ranged.tscn` | Cena do inimigo `ranged`. |
| `jogo/scripts/world/room_validator.gd` | Modificado para consumir a fábrica. |

Os arquivos vivem na branch `game` (implementação introduzida em `8dd6aa1` por Breno Lucena Cordeiro).

### Código

```gdscript
## Factory Method Pattern
## Centraliza a criação de inimigos. Adicione novos tipos aqui.
class_name EnemyFactory
extends Node

const ENEMY_SCENES: Dictionary = {
    &"basic":  preload("res://scenes/enemies/enemy.tscn"),
    &"ranged": preload("res://scenes/enemies/enemy_ranged.tscn"),
}

static func create(type: StringName) -> CharacterBody2D:
    if not ENEMY_SCENES.has(type):
        push_error("EnemyFactory: tipo desconhecido '%s'" % type)
        return null

    var enemy_scene: PackedScene = ENEMY_SCENES[type]
    var enemy: CharacterBody2D = enemy_scene.instantiate()

    match type:
        &"basic":  pass
        &"ranged": pass

    return enemy
```

### Tipos suportados

| Chave | Cena |
|---|---|
| `&"basic"` | `enemy.tscn` |
| `&"ranged"` | `enemy_ranged.tscn` |

O `match` no final permite configurar atributos específicos por tipo após a instanciação (atualmente todos os tipos usam os valores padrão da própria cena).

## 3. Funcionamento

```gdscript
# Antes: instanciação direta, espalhada pelo código
var enemy = load("res://scenes/enemies/enemy.tscn").instantiate()

# Depois: uma chamada uniforme
var basic  = EnemyFactory.create(&"basic")
var ranged = EnemyFactory.create(&"ranged")
```

Para adicionar um novo tipo, basta:

1. Criar a cena `.tscn` correspondente.
2. Adicionar uma entrada em `ENEMY_SCENES` com a chave e o `preload(...)`.
3. (Opcional) Adicionar uma branch no `match` para configurar atributos específicos.

Nenhum código consumidor precisa mudar.

### Benefícios

- **Centralização:** toda criação de inimigos passa por um único ponto.
- **Extensibilidade:** novo tipo = nova linha no dicionário, sem alterar o código que consome.
- **Desacoplamento:** o consumidor (ex: `room_validator.gd`) não conhece os detalhes do inimigo nem o caminho das cenas.
- **Validação:** tipo desconhecido emite `push_error` em vez de causar `null` silencioso.

## 4. Metodologia

A construção desta documentação seguiu três etapas, tomando como fonte primária o código original em `8dd6aa1`:

1. **Levantamento da Factory.** Leitura de `jogo/scripts/enemies/enemy_factory.gd` para mapear o dicionário `ENEMY_SCENES`, a assinatura estática de `create(type: StringName) -> CharacterBody2D` e o ponto de extensão via `match`.

2. **Trace do consumo.** Confirmação de que `room_validator.gd` foi modificado para usar `EnemyFactory.create(...)` em vez de `load(...).instantiate()` direto.

3. **Identificação dos benefícios.** Análise comparativa do código antes e depois para extrair os ganhos de centralização, extensibilidade e validação de tipos.

**Ferramentas utilizadas:** Godot 4 (leitura do código), Git (recuperação do commit `8dd6aa1`), Markdown + Docsify (apresentação).
