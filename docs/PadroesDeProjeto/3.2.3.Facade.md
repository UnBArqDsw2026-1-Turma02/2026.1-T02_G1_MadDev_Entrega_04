# 3.2.1. Facade (Estrutural)

## 1. Introdução

O **Facade** é um padrão estrutural do GoF que **fornece uma interface unificada para um conjunto de subsistemas**, escondendo a complexidade interna por trás de chamadas de alto nível. No projeto MadDev, a Facade `GameFacade` é um Autoload que centraliza o acesso aos demais autoloads (`AudioManager`, `GameManager`, `GameMediator` → `SignalBus`) — o código de gameplay chama `GameFacade.play_sound(...)` em vez de saber qual subsistema responde pelo quê.

### Quando usar

- Quando vários subsistemas precisam ser usados juntos em sequências repetidas.
- Quando o código de aplicação não deve conhecer a estrutura interna dos subsistemas.
- Quando se quer uma camada para evoluir/refatorar os subsistemas sem quebrar consumidores.

## 2. Implementação no Projeto

### Arquivos

| Arquivo | Papel |
|---|---|
| `jogo/scripts/autoloads/game_facade.gd` | A Facade — Autoload `GameFacade` em `project.godot`. |
| `jogo/scripts/ui/main_menu.gd` | Refatorado para consumir a Facade. |
| `jogo/project.godot` | Registro do autoload `GameFacade`. |

Os arquivos vivem na branch `game`.

### Estrutura

```
GameFacade  (autoload)
    │
    ├──>  AudioManager     (play_sound, play_music, stop_music)
    ├──>  GameManager      (start_run, end_run, toggle_pause, award_points, ...)
    └──>  GameMediator     (eventos → SignalBus)
```

A Facade **não tem estado próprio** — é apenas uma camada de delegação. Cada método público encaminha para o autoload correto.

### Código (recortes)

```gdscript
## Facade Pattern
## Interface simplificada para os subsistemas do jogo (Audio, Game, Signals).
## Registrado como Autoload com nome "GameFacade" no project.godot.
extends Node

# --- Áudio (AudioManager) ---
func play_sound(sfx_name: String) -> void:
    AudioManager.play_sfx_by_name(sfx_name)

func play_music(music_name: String) -> void:
    AudioManager.play_music_by_name(music_name)

func stop_music() -> void:
    AudioManager.stop_music()

# --- Jogo (GameManager) ---
func start_run() -> void:
    GameManager.start_run()

func reset_game_state() -> void:
    GameManager.reset_run()

func end_run(victory: bool) -> void:
    GameManager.end_run(victory)

func toggle_pause() -> void:
    GameManager.toggle_pause()

func award_points(amount: int) -> void:
    GameManager.add_score(amount)
```

### Métodos públicos por subsistema

| Subsistema | Métodos da Facade | Delega para |
|---|---|---|
| **Áudio** | `play_sound(sfx)`, `play_music(name)`, `stop_music()` | `AudioManager` |
| **Jogo** | `start_run()`, `reset_game_state()`, `end_run(victory)`, `toggle_pause()`, `award_points(n)` | `GameManager` |
| **Eventos** | (delega ao `GameMediator`, que ponteia ao `SignalBus`) | `GameMediator` |

> Observação importante: a seção de eventos da Facade encaminha para o **Mediator**, não direto para o `SignalBus`. Isso preserva o desacoplamento entre quem emite o evento e quem o escuta — ver [Mediator (3.3.2)](/PadroesDeProjeto/3.3.GoFsComportamentais.md) (em branches que documentam esse padrão).

## 3. Funcionamento

Antes da Facade, código de gameplay teria que saber de qual autoload chamar cada coisa:

```gdscript
# Sem Facade — código precisa conhecer a estrutura interna
func _on_player_pickup_item() -> void:
    AudioManager.play_sfx_by_name("pickup")
    GameManager.add_score(10)
    GameMediator.notify(self, GameMediator.EVENT_ITEM_PICKED_UP, {})
```

Com a Facade, a chamada fica plana e expressiva:

```gdscript
# Com Facade — uma única interface
func _on_player_pickup_item() -> void:
    GameFacade.play_sound("pickup")
    GameFacade.award_points(10)
    # eventos via GameFacade encaminham ao Mediator
```

### Benefícios

- **Acoplamento mínimo:** o consumidor depende só de `GameFacade`, não dos autoloads internos.
- **Refatoração segura:** trocar `AudioManager` por outro sistema de áudio só requer atualizar a Facade.
- **Curva de aprendizagem reduzida:** novo membro do time só precisa conhecer a Facade para tarefas comuns.
- **Documentação concentrada:** cada método tem um comentário `##` indicando seu propósito, que aparece no editor do Godot.

## 4. Metodologia

A construção desta documentação seguiu três etapas, tomando como fonte primária o código da branch `game`:

1. **Identificação dos subsistemas envolvidos.** Leitura dos autoloads em `jogo/scripts/autoloads/` (`AudioManager`, `GameManager`, `GameMediator`) para mapear quais responsabilidades a Facade abstrai.

2. **Análise de `game_facade.gd`.** Leitura dos métodos públicos, agrupando-os por subsistema (Áudio, Jogo, Eventos) e confirmando que cada um é apenas uma chamada de delegação — a Facade não acumula estado.

3. **Trace do consumo refatorado.** Verificação no `main_menu.gd` de que chamadas que antes iam direto aos autoloads agora passam pela Facade, e do registro do `GameFacade` em `project.godot`.

**Ferramentas utilizadas:** Godot 4 (leitura do código e validação do registro de Autoload), Git (leitura da branch `game`), Markdown + Docsify (apresentação).
