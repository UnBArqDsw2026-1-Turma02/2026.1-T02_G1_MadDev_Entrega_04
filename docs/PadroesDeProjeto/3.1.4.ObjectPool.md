# 3.1.3. Object Pool — Pool de Projéteis (Criacional)

## 1. Introdução

O **Object Pool** é um padrão criacional que **reutiliza um conjunto pré-alocado de objetos** em vez de criar e destruir instâncias continuamente. No projeto MadDev, projéteis são pré-instanciados no `_ready()` do pool e desativados (não destruídos) ao terminar — quando um inimigo `ranged` dispara, o pool devolve um projétil inativo já existente.

> Nota: o **Object Pool** **não consta** no catálogo original do GoF (Gamma et al., 1994); é um padrão criacional consolidado pela comunidade de desenvolvimento de jogos, popularizado pelo livro *Game Programming Patterns* de Robert Nystrom.

### Quando usar

- Quando criar e destruir instâncias é frequente e caro (ex: projéteis, partículas, inimigos descartáveis).
- Quando o churn de nós degrada performance ou causa picos de garbage collection.
- Quando o número máximo simultâneo de instâncias é conhecido ou estimável.

## 2. Implementação no Projeto

### Arquivos

| Arquivo | Papel |
|---|---|

Os arquivos vivem na branch `game`.

### ProjectilePool

```gdscript
class_name ProjectilePool
extends Node

@export var projectile_scene: PackedScene = preload("res://scenes/projectiles/projectile.tscn")
@export var pool_size: int = 10
@export var auto_expand: bool = true

var _pool: Array[Area2D] = []

func _ready() -> void:
    add_to_group("projectile_pool")
    _preload_projectiles()

func _preload_projectiles() -> void:
    for i in range(pool_size):
        var projectile = _create_projectile()
        _pool.append(projectile)
        add_child(projectile)
    print("ProjectilePool: ", pool_size, " projéteis pré-criados")

func _disable_projectile(projectile: Area2D) -> void:
    projectile.process_mode = Node.PROCESS_MODE_DISABLED
    projectile.visible = false
    projectile.set_process(false)
    if projectile.has_method("set_monitoring"):
        projectile.set_monitoring(false)
        projectile.set_monitorable(false)
```

Configuração via `@export`:

| Parâmetro | Padrão | Descrição |
|---|---|---|
| `projectile_scene` | `projectile.tscn` | Cena base do projétil. |
| `pool_size` | 10 | Quantos projéteis pré-criar no `_ready()`. |
| `auto_expand` | `true` | Se `true`, expande o pool quando todos estão em uso. |

### ProjectileBase — par `enable/disable`

```gdscript
extends Area2D

@export var speed: float = 300.0
@export var damage: int = 10
@export var lifetime: float = 3.0

var direction: Vector2 = Vector2.RIGHT
var _elapsed: float = 0.0
var _pool: Node = null

func enable(spawn_position: Vector2, spawn_direction: Vector2) -> void:
    _shooter = null
    global_position = spawn_position
    direction = spawn_direction.normalized()
    _elapsed = 0.0
    show()
    set_process(true)
    monitoring = true
    monitorable = true

func disable() -> void:
    # Guard contra double-call: pool desativa via process_mode antes de chamar disable().
    if process_mode == Node.PROCESS_MODE_DISABLED:
        return
    hide()
    set_process(false)
    monitoring = false
    monitorable = false
    if _pool != null and _pool.has_method("return_projectile"):
        _pool.return_projectile(self)
```

O guard `if process_mode == Node.PROCESS_MODE_DISABLED: return` em `disable()` protege contra colisão e timeout chamando `disable()` simultaneamente.

## 3. Funcionamento

Em vez de:

```gdscript
# Sem pool: cria e destrói a cada disparo
var bullet = preload("res://scenes/projectiles/projectile.tscn").instantiate()
add_child(bullet)
# ... e mais tarde:
bullet.queue_free()
```

Com pool:

```gdscript
# Com pool: reutiliza instâncias existentes
var pool := get_tree().get_first_node_in_group("projectile_pool") as ProjectilePool
pool.get_projectile(global_position, direction, self)
# Quando colide ou expira o lifetime:
#   projectile.disable() devolve a instância ao pool
```

### Resultado dos testes

| Momento | Total | Ativos | Disponíveis |
|---|---|---|---|
| Início | 10 | 0 | 10 |
| Após 5 disparos | 10 | 5 | 5 |
| Após expirarem | 10 | 0 | 10 |

O total não cresce porque os projéteis são reutilizados — eles passam de "ativo" para "disponível" via `disable()` em vez de serem destruídos.

### Benefícios

- **Sem alocação contínua:** projéteis são instanciados uma única vez no `_ready()`.
- **Sem garbage collection visível:** `queue_free()` não é chamado durante o gameplay.
- **Performance previsível:** custo de "disparar" é constante (`get_projectile` + reposicionamento).
- **Crescimento controlado:** `auto_expand` evita pré-alocação excessiva, mas permite cobrir picos.

## 4. Metodologia

A construção desta documentação seguiu três etapas, tomando como fonte primária o código da branch `game`:

1. **Análise do ciclo de vida do pool.** Leitura de `projectile_pool.gd` para mapear o `_preload_projectiles()` no `_ready()`, a estrutura interna `_pool: Array[Area2D]` e os métodos auxiliares `_create_projectile`, `_disable_projectile`, `_enable_projectile`.

2. **Trace de `enable`/`disable` no projétil.** Leitura de `projectile_base.gd` para identificar a interface pública de reuso (`enable(pos, dir)` e `disable()`) e o guard contra double-call em colisão simultânea com expiração de `lifetime`.

3. **Inspeção da configuração via `@export`.** Identificação dos parâmetros expostos no inspetor (`pool_size`, `auto_expand`, `projectile_scene`) e da semântica do `auto_expand` quando todos os projéteis estão em uso.

**Ferramentas utilizadas:** Godot 4 (leitura do código e validação em runtime), Git (leitura da branch `game`), Markdown + Docsify (apresentação).
