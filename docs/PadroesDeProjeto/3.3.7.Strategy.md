# 3.3.3. Strategy (Comportamental)
 
## 1. Introdução
 
O **Strategy** é um padrão comportamental do GoF que **define uma família de algoritmos, encapsula cada um deles e os torna intercambiáveis**, permitindo que o algoritmo varie independentemente dos clientes que o utilizam. No projeto MadDev, o Strategy resolve o problema de inimigos com comportamentos de movimento distintos: em vez de subclasses de `EnemyBase` para cada tipo de pathfinding, cada inimigo possui um `@export var move_strategy: MoveStrategy` que pode ser trocado pelo inspetor do Godot em tempo de design ou por código em tempo de execução. Isso mantém `enemy_base.gd` estável mesmo com novos algoritmos de movimento.
 
### Quando usar
 
- Quando diferentes variantes de um algoritmo devem ser intercambiáveis em tempo de execução.
- Quando há múltiplas subclasses que diferem apenas no comportamento de um método específico.
- Quando se quer separar o algoritmo do objeto que o utiliza, facilitando testes e extensão.

## 2. Implementação no Projeto
 
### Arquivos
 
| Arquivo | Papel |
|---|---|
| `jogo/scripts/enemies/strategies/move_strategy.gd` | Estratégia base — define a interface `get_velocity(enemy, target)`. |
| `jogo/scripts/enemies/strategies/direct_chase_strategy.gd` | Perseguição direta em linha reta, sem desvio de obstáculos. |
| `jogo/scripts/enemies/strategies/navigation_strategy.gd` | Pathfinding via `NavigationAgent2D`, desvia de obstáculos. |
| `jogo/scripts/enemies/strategies/patrol_strategy.gd` | Patrulha cíclica por pontos fixos, ignora o alvo. |
| `jogo/scripts/enemies/enemy_base.gd` | Modificado — usa `move_strategy` em vez de `_get_move_direction()` herdado. |
 
Os arquivos foram introduzidos na branch `feat/9-strategy-pathfinding`.
 
### MoveStrategy — interface base
 
```gdscript
extends Resource
class_name MoveStrategy
 
## Calcula e retorna o vetor de velocidade para este frame.
## enemy.move_speed está disponível para uso interno.
func get_velocity(enemy: CharacterBody2D, target: Node2D) -> Vector2:
    return Vector2.ZERO
```
 
### Estratégias implementadas
 
| Estratégia | Algoritmo | Requisitos |
|---|---|---|
| `DirectChaseStrategy` | Linha reta ao alvo: `(target.pos - enemy.pos).normalized() * speed` | Nenhum |
| `NavigationStrategy` | `NavigationAgent2D.get_next_path_position()` | Filho `NavigationAgent2D` no inimigo |
| `PatrolStrategy` | Cicla por `patrol_points: Array[Vector2]` | Lista de posições configurada |
 
### enemy_base.gd — integração
 
```gdscript
@export var move_strategy: MoveStrategy = null
 
func _physics_process(_delta: float) -> void:
    if _is_dead:
        return
 
    if move_strategy != null:
        # Strategy Pattern: delega ao algoritmo ativo
        velocity = move_strategy.get_velocity(self, _target)
    else:
        # Fallback por herança — mantém compatibilidade com subclasses existentes
        velocity = _get_move_direction() * move_speed
 
    move_and_slide()
```
 
### PatrolStrategy — configuração
 
```gdscript
@export var patrol_points: Array[Vector2] = []
@export var patrol_threshold: float = 12.0
var _current_idx: int = 0
 
func get_velocity(enemy: CharacterBody2D, _target: Node2D) -> Vector2:
    if patrol_points.is_empty():
        return Vector2.ZERO
    var destination := patrol_points[_current_idx]
    var to_dest := destination - enemy.global_position
    if to_dest.length() <= patrol_threshold:
        _current_idx = (_current_idx + 1) % patrol_points.size()
    return to_dest.normalized() * enemy.move_speed
```

## 3. Funcionamento
 
```gdscript
# No inspetor do Godot: atribuir diretamente (sem código)
# inimigo.move_strategy = DirectChaseStrategy.new()
 
# Por código — trocar estratégia em tempo de execução:
var inimigo_patrulha: EnemyBase = $Guarda
var patrulha := PatrolStrategy.new()
patrulha.patrol_points = [Vector2(100, 100), Vector2(300, 100), Vector2(300, 300)]
inimigo_patrulha.move_strategy = patrulha
 
# Ao detectar o jogador, trocar para perseguição:
inimigo_patrulha.move_strategy = DirectChaseStrategy.new()
```
 
A `NavigationStrategy` emite um `push_warning()` caso o inimigo não possua um filho `NavigationAgent2D`, caindo de volta para linha reta — sem crash silencioso.
 
### Benefícios
 
- **Composição sobre herança:** `move_strategy` é um `@export`, trocável no inspetor sem subclasses.
- **Backward compatible:** `_get_move_direction()` permanece como fallback para inimigos que não usam Strategy.
- **Extensibilidade:** nova estratégia = novo Resource `extends MoveStrategy`, sem alterar `enemy_base.gd`.
- **Falha segura:** `NavigationStrategy` tem fallback para linha reta quando sem `NavigationAgent2D`.

## 4. Metodologia
 
A construção desta documentação seguiu três etapas, tomando como fonte primária o código da branch `feat/9-strategy-pathfinding`:
 
1. **Levantamento das estratégias.** Leitura de `move_strategy.gd` (interface base) e dos 3 concretos para mapear a assinatura `get_velocity(enemy: CharacterBody2D, target: Node2D) -> Vector2` e as diferenças de requisito (nenhum / NavigationAgent2D / patrol_points) entre as implementações.
2. **Análise da integração em enemy_base.** Verificação do bloco `if move_strategy != null` que aplica Strategy por composição, e do `else` que preserva o comportamento legado via `_get_move_direction()`, garantindo que inimigos concretos existentes não precisem ser modificados.
3. **Trace da NavigationStrategy.** Confirmação do `get_node_or_null("NavigationAgent2D")` com `push_warning()` e fallback para linha reta, e da atualização de `nav_agent.target_position` a cada frame para que o pathfinding siga um alvo em movimento.
**Ferramentas utilizadas:** Godot 4 (validação de `NavigationAgent2D.get_next_path_position()`), Git (branch `feat/9-strategy-pathfinding`), Markdown + Docsify (apresentação).
