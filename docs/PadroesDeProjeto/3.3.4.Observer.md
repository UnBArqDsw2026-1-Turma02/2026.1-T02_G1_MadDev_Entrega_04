# 3.3.4. Observer (Comportamental)

## 1. Introdução

O **Observer** é um padrão comportamental do GoF que **define uma dependência um-para-muitos** entre objetos, de modo que quando um objeto muda de estado, todos os seus dependentes são notificados automaticamente. No projeto MadDev, o Observer é aplicado no sistema de conquistas: o `AchievementManager` se conecta a sinais do `SignalBus` (`enemy_died`, `item_picked_up`, `room_cleared`, `score_changed`, `player_health_changed`, `run_started`) e desbloqueia conquistas quando as condições — encapsuladas como `Callable` em cada `Achievement` — são satisfeitas.

### Quando usar

- Quando vários sistemas precisam reagir ao mesmo evento sem que o emissor conheça os receptores.
- Quando a lista de receptores muda em runtime ou depende de configuração.
- Quando se quer evitar acoplamento direto entre quem dispara o evento e quem responde a ele.

### Estrutura

```
SignalBus (Subject)
  │
  ├── enemy_died ──────────┐
  ├── item_picked_up ──────┤
  ├── room_cleared ────────┼──▶ AchievementManager (Observer)
  ├── score_changed ───────┤        │
  ├── player_health_changed┤        ├─ atualiza _stats
  └── run_started ─────────┘        └─ percorre Array[Achievement]
                                       │
                                       └─ avalia condition.call(_stats)
                                          │
                                          └─ emite achievement_unlocked
```

## 2. Implementação no Projeto

### Arquivos

| Arquivo | Papel no padrão |
|---|---|
| `jogo/scripts/autoloads/signal_bus.gd` | **Subject** — barramento de sinais que dispara os eventos |
| `jogo/scripts/autoloads/achievement_manager.gd` | **ConcreteObserver** — autoload que escuta o `SignalBus` e desbloqueia conquistas |
| `jogo/scripts/resources/achievement.gd` | **Resource** com `id`, `name`, `description`, `unlocked` e `condition: Callable` |

Os arquivos vivem na branch `game`.

### Achievement (Resource)

```gdscript
extends Resource
class_name Achievement

@export var id: StringName = &""
@export var name: String = ""
@export var description: String = ""
@export var unlocked: bool = false

var event: StringName = &""
var condition: Callable

func setup(new_id, new_name, new_description, new_event, new_condition) -> Achievement:
    id = new_id
    name = new_name
    description = new_description
    event = new_event
    condition = new_condition
    unlocked = false
    return self

func can_unlock(data: Dictionary) -> bool:
    if unlocked or not condition.is_valid():
        return false
    return condition.call(data)
```

Cada conquista guarda **a sua própria regra** como `Callable`, eliminando a necessidade de `switch`/`if` por conquista no Manager.

### AchievementManager (Observer)

```gdscript
extends Node

var achievements: Array[Achievement] = []
var _stats: Dictionary = {
    "enemies_defeated": 0,
    "items_collected": 0,
    "rooms_cleared": 0,
    "score": 0,
    "player_health": 0,
    "player_max_health": 0,
}

func _ready() -> void:
    _create_achievements()
    _connect_signal_bus()

func _connect_signal_bus() -> void:
    SignalBus.run_started.connect(_on_run_started)
    SignalBus.enemy_died.connect(_on_enemy_died)
    SignalBus.item_picked_up.connect(_on_item_picked_up)
    SignalBus.room_cleared.connect(_on_room_cleared)
    SignalBus.player_health_changed.connect(_on_player_health_changed)
    SignalBus.score_changed.connect(_on_score_changed)

func _evaluate_achievements_for_event(event: StringName) -> void:
    for achievement in achievements:
        if achievement.event == event and achievement.can_unlock(_stats):
            achievement.unlocked = true
            SignalBus.achievement_unlocked.emit(achievement)
```

### Conquistas configuradas

| ID | Nome | Evento gatilho | Condição |
|---|---|---|---|
| `&"first_blood"` | Primeiro Sangue | `enemy_died` | `enemies_defeated >= 1` |
| `&"enemy_hunter"` | Caçador de Bugs | `enemy_died` | `enemies_defeated >= 5` |
| `&"collector"` | Colecionador | `item_picked_up` | `items_collected >= 3` |
| `&"first_room"` | Sala Dominada | `room_cleared` | `rooms_cleared >= 1` |
| `&"survivor"` | Sobrevivente | `room_cleared` | `rooms_cleared >= 1 and player_health > 0` |
| `&"high_score"` | Nota Máxima | `score_changed` | `score >= 100` |

Cada uma é criada via `Achievement.new().setup(id, name, description, event, Callable)` — a `Callable` referencia o método privado correspondente no Manager (`_has_defeated_first_enemy`, `_has_cleared_room_alive`, etc.).

## 3. Funcionamento

```gdscript
# Em algum lugar do gameplay (ex: enemy_base.gd quando inimigo morre):
SignalBus.enemy_died.emit(self)

# AchievementManager (já registrado) recebe:
func _on_enemy_died(_enemy: Node) -> void:
    _stats["enemies_defeated"] += 1
    _evaluate_achievements_for_event(&"enemy_died")
    # Se passar de 1 → desbloqueia "Primeiro Sangue"
    # Se passar de 5 → desbloqueia "Caçador de Bugs"
```

Quando uma conquista é desbloqueada, o Manager emite `SignalBus.achievement_unlocked` com o `Achievement` como payload. O HUD (`hud_abstraction.gd`) também é Observer desse sinal e mostra uma notificação temporária no canto da tela.

### Atalhos de debug

Em build de desenvolvimento (`OS.is_debug_build()`), o Manager intercepta input para facilitar testes manuais:

| Tecla | Efeito |
|---|---|
| `K` | Emite `SignalBus.enemy_died` (simula inimigo derrotado) |
| `I` | Emite `SignalBus.item_picked_up` (simula item coletado) |
| `R` | Emite `SignalBus.room_cleared` (simula sala completada) |
| `P` | Chama `GameManager.add_score(100)` |

### Benefícios

- **Acoplamento mínimo:** quem emite o sinal não conhece o Manager. O Manager não conhece quem emite.
- **Regras encapsuladas:** cada conquista carrega sua própria `Callable` — adicionar uma nova é só uma linha em `_create_achievements()`, sem `switch` no Manager.
- **Reset trivial:** `_on_run_started` chama `reset_achievements()` zerando `_stats` e marcando todas como `unlocked = false`.
- **Composição com outros padrões:** o Manager funciona em conjunto com o `Mediator` (via `SignalBus`), demonstrando que vários padrões podem coexistir no mesmo sistema.

## 4. Metodologia

A construção desta documentação seguiu três etapas, tomando como fonte primária o código da branch `game`:

1. **Mapeamento dos sinais do Subject.** Leitura de `achievement_manager.gd` (linhas de `_connect_signal_bus()`) para listar os 6 sinais escutados do `SignalBus` e os handlers correspondentes.

2. **Análise das conquistas via `Callable`.** Inspeção de `_create_achievements()` e dos métodos `_has_*` para mapear evento gatilho, condição e dados consultados em `_stats` — base para a tabela de conquistas.

3. **Trace do emit de `achievement_unlocked`.** Verificação de que `_evaluate_achievements_for_event` é chamado em cada handler de evento, e de que o `Achievement` desbloqueado é repassado ao HUD via `SignalBus.achievement_unlocked.emit(achievement)`.

**Ferramentas utilizadas:** Godot 4 (execução com os atalhos de debug `K`/`I`/`R`/`P`), Git (leitura da branch `game`), Markdown + Docsify (apresentação).
